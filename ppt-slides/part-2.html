Introduction

Git: The Project Tracker

A decentralized version control system that tracks every change made to your files. It allows you to travel back in time to previous versions of your project.

Setup

Identify Yourself

Before you start, Git needs to know who is making the changes. This information is saved in your project history.

TERMINAL
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

# To check your settings:
git config --list
Tip: The --global flag ensures these settings apply to all repositories on your computer.
Concepts

The Three Stages

1. Working Dir
Your local folder where files exist.
2. Staging (Index)
The review section for unpublished work.
3. Repository
The final database of saved snapshots.
Deployment

Pushing: The First Time

Use these commands to link your local code to a brand new GitHub repository.

TERMINAL
cd path/to/your/project
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/yourusername/repository-name.git
git push -u origin main
Workflow

Pushing: Subsequent Updates

After the first time, your update loop is fast and simple.

TERMINAL
git add .
git commit -m "Initial commit"
git push
Collaboration

Cloning & Contributing

TERMINAL
git clone <repository-url>
cd <repository-folder>
git branch -a
git checkout -b my-feature-branch
Advanced Flow

Sync & Pull Requests

TERMINAL
git pull --rebase origin <branch-name>
# Edit files...
git add .
git commit -m "Describe your changes"
git push origin my-feature-branch

Finally: Open a Pull Request on GitHub to merge your work.

Workflow: Always pull --rebase during development to stay updated with the main team.
