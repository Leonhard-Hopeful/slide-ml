Section 01

Data Acquisition

The foundation of any ML project. Before writing a single line of model code, you must determine if your data can be trusted.

Acquisition

Identifying Sources

  • First-Party Data: Internal databases, CRM, logs (Highest trust).
  • Third-Party Data: APIs, Marketplace data (Kaggle, AWS Data Exchange).
  • Unstructured Data: Web scraping, PDF extraction (High maintenance).
Acquisition

The Pandas Entrypoint

PYTHON
import pandas as pd

# Professional loading with explicit parameters
df = pd.read_csv('raw_data.csv', 
                 na_values=['?', 'n/a'],
                 low_memory=False,
                 parse_dates=['timestamp'])
Acquisition

The Initial Inspection

Never assume the data matches the documentation. Verify immediately:

df.info()      # Check memory usage and non-null counts
df.describe()  # Check for impossible values (e.g. age = -1)
df.nunique()   # Check cardinality of categories

Spotting Selection Bias

Snapshotting Raw Data

Always preserve the "Original" state before transformations. This allows you to trace errors back to the source.

Section 02

Cleaning & Preprocessing

Data cleaning is not a chore; it is feature engineering. How you handle missing data determines how your model "sees" the world.

Cleaning

Handling Missing Values

Three professional approaches:

  • Dropping: Only if missingness is completely random.
  • Imputation (Median/Mode): Standard for small gaps.
  • Predictive Imputation: Using other columns to estimate the missing value.
Cleaning

Imputation Logic

# Professional imputation using Scikit-Learn
from sklearn.impute import SimpleImputer

imputer = SimpleImputer(strategy='median')
df['age'] = imputer.fit_transform(df[['age']])
Cleaning

The Outlier Dilemma

Is an outlier an error (typo) or a discovery (a high-value whale customer)?

Cleaning

Z-Score Filtering

from scipy import stats
import numpy as np

z_scores = np.abs(stats.zscore(df['salary']))
df_clean = df[z_scores < 3] # Keep data within 3 standard deviations
Preprocessing

Encoding Variables

Converting labels into numbers for ML models:

# One-Hot Encoding for Nominal data
df = pd.get_dummies(df, columns=['city'])

# Label Encoding for Ordinal data
df['size'] = df['size'].map({'Small': 1, 'Medium': 2, 'Large': 3})
Preprocessing

Normalizing Scales

Models like SVM or KNN fail if one feature is 1,000x larger than another.

from sklearn.preprocessing import StandardScaler
scaler = StandardScaler()
df_scaled = scaler.fit_transform(df)
Preprocessing

Clean Names

Professional tip: Lowercase, snake_case, and descriptive.

df.columns = [c.lower().replace(' ', '_') for c in df.columns]
Cleaning

The Hidden Duplicate

Sometimes duplicates aren't identical rows; they are the same user with two different IDs.

Cleaning Checklist

Is it Model Ready?

  • [ ] No NaN values remaining.
  • [ ] Numeric types correctly cast.
  • [ ] Categorical data encoded.
  • [ ] Features scaled and normalized.
Section 03

Exploratory Data Analysis (EDA)

EDA is where the analyst becomes a detective. We look for the patterns that matter and the correlations that lie.

EDA

Three Main Questions

  1. What is the central tendency (Mean/Median)?
  2. What is the spread (Variance/IQR)?
  3. Are there linear or non-linear relationships?
EDA

Visualizing Distribution

import seaborn as sns
import matplotlib.pyplot as plt

sns.displot(df['price'], kde=True)
plt.title('Price Distribution')
EDA

Relationship Analysis

sns.regplot(x='advertising_spend', y='sales', data=df)
plt.show()
EDA

The Heatmap Warning

If two features have a correlation > 0.9, you have redundant info. This can confuse your model.

sns.heatmap(df.corr(), annot=True)
EDA

Pandas Pivots

# See average revenue by Region and Product Category
report = df.pivot_table(index='region', 
                        columns='category', 
                        values='revenue', 
                        aggfunc='mean')
EDA

Deep Aggregates

df.groupby('customer_segment')['clv'].agg(['mean', 'max', 'count'])
EDA

Why Visualize?

Four datasets can have identical Mean and Variance but look completely different when plotted. Always plot your data.

EDA

Sanity Checks

Does the data make sense? If your retail data shows a surge in sales of "Summer Shorts" in December, check if the data source is from the Southern Hemisphere.
EDA Summary

Hypothesis Generation

By the end of EDA, you should have a list of features you suspect will be most predictive for your ML model.

Section 04

Professional Data Visualization

Visualization is not decoration. It is a communication protocol. A bad chart can lead to a bad business decision.

Visualization

The Hierarchy of Perception

The human eye is best at comparing Position (Scatter Plots) and Length (Bar Charts). It is worst at comparing Angles (Pie Charts).

Rule: Avoid Pie Charts in professional reports. Use Horizontal Bar Charts instead.
Visualization

Strategic Color

  • Sequential: Low to high (Light blue to Dark blue).
  • Diverging: Critical points (Red to White to Green).
  • Qualitative: Categorical (Different colors for Apple, Orange, Banana).
Visualization

Standardizing Style

plt.style.use('seaborn-v0_8-whitegrid')
plt.rcParams['figure.figsize'] = (10, 6)
Visualization

Multi-Panel Figures

fig, axes = plt.subplots(1, 2)
sns.boxplot(ax=axes[0], y=df['age'])
sns.violinplot(ax=axes[1], y=df['age'])
Visualization

The Unlabeled Sin

An unlabelled axis is a lie. Every chart must have: Title, X-label, Y-label, and Legend.
Visualization

Managing Density

Too many points on a scatter plot? Use Alpha (transparency) or Hexbin plots.

plt.scatter(x, y, alpha=0.1)
Visualization

Enter Plotly

For stakeholder dashboards, interactive plots allow them to hover and see exact values.

import plotly.express as px
fig = px.scatter(df, x='gdp', y='life_exp', hover_name='country')
fig.show()
Visualization

Quick Picker

  • Trends: Line Chart.
  • Comparisons: Bar Chart.
  • Relationships: Scatter Plot.
  • Composition: Stacked Area.
Visual Checklist

The Squint Test

Squint your eyes at the chart. If the main message (upward trend, big outlier) isn't visible, the chart is too cluttered.

Section 05

From Numbers to Narrative

Data doesn't speak for itself. You are the translator. You must turn statistical outputs into business value.

Narrative

The Stakeholder Filter

Executives don't care about p-values or R-squared. They care about:

  • Risk reduction.
  • Revenue growth.
  • Time savings.
Narrative

The Professional Caveat

"The data suggests..." vs "The data proves..."

Always distinguish between a correlation and a causal link that requires an A/B test to confirm.

Narrative

The Executive Summary

B.L.U.F: Bottom Line Up Front. Put the answer in the first paragraph, then provide the evidence.

Narrative

Communicating Confidence

If you have a 60% confidence recommendation, say so explicitly. Do not present estimates as certainties.

Narrative

Ending with Action

Never end an analysis with "That's interesting." End with: "Based on this, we should [X]."
