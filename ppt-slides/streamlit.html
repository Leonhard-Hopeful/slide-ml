Why Streamlit for ML?

As ML engineers we spend most of our time in notebooks. But notebooks can't be shared with stakeholders, product teams, or clients. Streamlit bridges that gap — it turns Python scripts into shareable web apps with minimal friction.

Your Model
.pkl / .pt / .h5
→
Streamlit App
app.py
→
Interactive UI
browser
→
Stakeholders
share link

The core mental model: Streamlit re-runs your entire script top-to-bottom every time a user interacts with a widget. That's it. No callbacks, no state management boilerplate, no JavaScript.

💡
This re-run model is a feature, not a bug. It makes reasoning about your app trivially simple. We'll learn how to cache expensive operations like model loading so this doesn't hurt performance.

Installation & First App

1
Create a virtual environment and install
bash
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install streamlit scikit-learn pandas numpy pillow
2
Create your first file — app.py
python — app.py
import streamlit as st

st.title("My First ML App 🤖")
st.write("Hello, Streamlit!")

name = st.text_input("What's your name?")
if name:
    st.success(f"Welcome, {name}! Let's build something cool.")
3
Run it
bash
streamlit run app.py
Your browser opens automatically at localhost:8501. Every time you save app.py, the app hot-reloads.

Input Widgets

Every widget returns a value you can use directly in Python. When the user changes a widget, the script re-runs and the new value is available.

st.slider()
Numeric range — great for thresholds, confidence cutoffs, hyperparameters.
st.selectbox()
Dropdown — choose model type, class label, feature set.
st.multiselect()
Multi-choice — select features to include in inference.
st.number_input()
Exact numeric entry — perfect for feature values.
st.text_input()
Free text — NLP tasks, prompts, labels.
st.checkbox()
Boolean toggle — show/hide extra info, enable preprocessing.
st.radio()
Exclusive choice — pick inference mode or model variant.
st.button()
Trigger action — run prediction, clear cache, download results.
python — widgets in action
# Each widget returns its current value immediately
threshold   = st.slider("Confidence Threshold", 0.0, 1.0, 0.5, step=0.05)
model_name  = st.selectbox("Model", ["Random Forest", "XGBoost", "Logistic Reg"])
feature_age = st.number_input("Age", min_value=0, max_value=120, value=30)
run_explain = st.checkbox("Show SHAP explanations")

# Values are just Python variables — use them anywhere
if st.button("🔍 Predict"):
    result = my_model.predict_proba([[feature_age, ...]])
    if result[0][1] >= threshold:
        st.success("Positive prediction ✅")
    else:
        st.error("Negative prediction ❌")

Layout & Structure

Use columns, sidebars, tabs, and expanders to organise your ML app cleanly.

python — layout primitives
# ── Sidebar (inputs live here, results in main area) ──
with st.sidebar():
    st.header("⚙️ Controls")
    model = st.selectbox("Model", ["v1", "v2"])
    threshold = st.slider("Threshold", 0.0, 1.0, 0.5)

# ── Columns ──
col1, col2, col3 = st.columns([2, 1, 1])   # relative widths
with col1:
    st.metric("Accuracy", "94.2%")
with col2:
    st.metric("Precision", "91.0%")
with col3:
    st.metric("Recall", "88.5%")

# ── Tabs ──
tab1, tab2, tab3 = st.tabs(["📊 Results", "📈 Metrics", "🔍 Explain"])
with tab1:
    st.write("Prediction results here")
with tab2:
    st.write("Confusion matrix, ROC curve")
with tab3:
    st.write("Feature importance / SHAP")

# ── Expander (hide verbose outputs) ──
with st.expander("🔎 See raw prediction scores"):
    st.json({"class_0": 0.12, "class_1": 0.88})
✅
Pro pattern: Put all user inputs in st.sidebar. Keep the main area for outputs. This mirrors how dashboards and ML tools are typically designed — controls on the left, results on the right.

Loading Models — The Right Way

Because Streamlit re-runs on every interaction, loading a model inside a plain function would reload it on every click. @st.cache_resource is the fix — it runs the function once and caches the result in memory for the entire session.

python — caching patterns
import streamlit as st
import joblib
import torch
from transformers import pipeline

# ── @st.cache_resource
# For: models, DB connections, API clients
# Lives as long as the server is running ──
@st.cache_resource
def load_sklearn_model():
    return joblib.load("model.pkl")

@st.cache_resource
def load_pytorch_model():
    model = MyNet()
    model.load_state_dict(torch.load("model.pth", map_location="cpu"))
    model.eval()
    return model

@st.cache_resource
def load_hf_pipeline():
    return pipeline("sentiment-analysis")

# ── @st.cache_data
# For: preprocessed datasets, DataFrames, API responses
# Tied to input arguments — re-runs if args change ──
@st.cache_data
def load_dataset(path):
    return pd.read_csv(path)

# ── Usage — model loads only ONCE ──
model = load_sklearn_model()
st.write("Model loaded!", type(model).__name__)
DecoratorUse ForCache Key
@st.cache_resourceModels, connections, heavy singletonsFunction identity
@st.cache_dataDataFrames, computed results, API callsFunction + arguments

Inference Flow Pattern

Here's the canonical pattern for every ML inference app. Follow this structure and your app will be clean and maintainable.

python — inference app template
import streamlit as st
import numpy as np
import joblib

# ─────────────────────────────────
# STEP 1 — Page config (must be first st call)
# ─────────────────────────────────
st.set_page_config(
    page_title="My ML App",
    page_icon="🤖",
    layout="wide"
)

# ─────────────────────────────────
# STEP 2 — Load model (cached)
# ─────────────────────────────────
@st.cache_resource
def load_model():
    return joblib.load("model.pkl")

model = load_model()

# ─────────────────────────────────
# STEP 3 — Collect user inputs
# ─────────────────────────────────
st.title("🏠 House Price Predictor")

with st.sidebar():
    st.header("Input Features")
    area    = st.number_input("Area (sqft)", 200, 10000, 1200)
    rooms   = st.slider("Bedrooms", 1, 10, 3)
    age     = st.number_input("House Age (years)", 0, 100, 10)
    predict = st.button("💰 Estimate Price", type="primary")

# ─────────────────────────────────
# STEP 4 — Preprocess & predict
# ─────────────────────────────────
if predict:
    X = np.array([[area, rooms, age]])         # build feature vector
    
    with st.spinner("Running inference..."):
        price = model.predict(X)[0]

# ─────────────────────────────────
# STEP 5 — Display results
# ─────────────────────────────────
    st.metric("Estimated Price", f"${price:,.0f}")
    st.balloons()   # 🎈 fun touch
⚠️
st.set_page_config() must be the very first Streamlit call in your script — before any other st. call. Putting it elsewhere raises an error.

For image classifiers and NLP models, users need to upload their own data. st.file_uploader returns a file-like object you can pass directly to PIL, pandas, or NumPy.

import pandas as pd
import numpy as np

)

if uploaded_img:
    img = Image.open(uploaded_img)

Project 1 — Tabular Classifier

🌸 Iris Species Classifier Beginner

Train a Random Forest on Iris, then build a Streamlit app where users adjust feature sliders to predict the species in real-time.

python — iris_app.py
import streamlit as st
import numpy as np
import plotly.express as px
import pandas as pd
from sklearn.datasets import load_iris
from sklearn.ensemble import RandomForestClassifier

st.set_page_config(page_title="Iris Classifier", page_icon="🌸", layout="wide")

@st.cache_resource
def train_model():
    iris = load_iris()
    clf = RandomForestClassifier(n_estimators=100, random_state=42)
    clf.fit(iris.data, iris.target)
    return clf, iris

model, iris = train_model()
CLASSES = iris.target_names

st.title("🌸 Iris Species Classifier")
st.markdown("Adjust the sliders to predict the iris species in real-time.")

# ── Inputs ──
with st.sidebar():
    st.header("🔧 Feature Inputs")
    sl = st.slider("Sepal Length (cm)", 4.0, 8.0, 5.4)
    sw = st.slider("Sepal Width (cm)",  2.0, 4.5, 3.0)
    pl = st.slider("Petal Length (cm)", 1.0, 7.0, 4.0)
    pw = st.slider("Petal Width (cm)",  0.1, 2.5, 1.2)

# ── Real-time prediction (no button needed!) ──
X = np.array([[sl, sw, pl, pw]])
probs = model.predict_proba(X)[0]
pred  = CLASSES[np.argmax(probs)]

# ── Show results ──
col1, col2 = st.columns([1, 2])

with col1:
    st.subheader("Prediction")
    st.metric("Species", pred.capitalize())
    st.metric("Confidence", f"{max(probs):.1%}")

with col2:
    prob_df = pd.DataFrame({
        "Species": CLASSES, "Probability": probs
    })
    fig = px.bar(prob_df, x="Species", y="Probability",
                color="Probability", color_continuous_scale="viridis",
                range_y=[0, 1])
    st.plotly_chart(fig, use_container_width=True)
🎯
Try it: Notice how there's no button — the prediction updates live as sliders move. This is possible because Streamlit re-runs on every widget change, and the model is cached so it's instant.

Project 2 — Image Classifier

🖼️ Image Classifier with Upload Intermediate

Accept user image uploads, preprocess for a CNN (PyTorch/TF), run inference, and display top-K predictions with a confidence bar chart.

python — image_classifier_app.py
import streamlit as st
import torch
import torchvision.transforms as T
import torchvision.models as models
from PIL import Image
import plotly.express as px
import pandas as pd
import json

st.set_page_config(page_title="Image Classifier", page_icon="🖼️", layout="wide")

@st.cache_resource
def load_model():
    model = models.mobilenet_v3_small(pretrained=True)
    model.eval()
    return model

@st.cache_data
def load_labels():
    with open("imagenet_labels.json") as f:
        return json.load(f)

TRANSFORM = T.Compose([
    T.Resize(256), T.CenterCrop(224),
    T.ToTensor(),
    T.Normalize([0.485, 0.456, 0.406], [0.229, 0.224, 0.225])
])

model = load_model()
labels = load_labels()

st.title("🖼️ Image Classifier")
top_k = st.sidebar.slider("Top-K Results", 3, 10, 5)

uploaded = st.file_uploader("Upload an image", type=["jpg", "png", "jpeg"])

if uploaded:
    img = Image.open(uploaded).convert("RGB")
    col1, col2 = st.columns([1, 2])
    
    with col1:
        st.image(img, caption="Input", use_column_width=True)
    
    with col2:
        with st.spinner("Classifying..."):
            tensor = TRANSFORM(img).unsqueeze(0)
            with torch.no_grad():
                out    = model(tensor)
                probs  = torch.softmax(out[0], dim=0)
                top_p, top_i = torch.topk(probs, top_k)
        
        results = pd.DataFrame({
            "Class": [labels[i.item()] for i in top_i],
            "Confidence": [p.item() for p in top_p]
        })
        st.subheader("Top Predictions")
        fig = px.bar(results, x="Confidence", y="Class",
                    orientation="h", range_x=[0, 1])
        st.plotly_chart(fig, use_container_width=True)

Project 3 — NLP Sentiment App

💬 Sentiment Analyzer Intermediate

Use a Hugging Face pipeline to analyze sentiment on free text or a batch of uploaded reviews, with an interactive results dashboard.

python — sentiment_app.py
import streamlit as st
import pandas as pd
import plotly.express as px
from transformers import pipeline

st.set_page_config(page_title="Sentiment Analyzer", page_icon="💬", layout="wide")

@st.cache_resource
def load_pipeline():
    return pipeline("sentiment-analysis",
                    model="distilbert-base-uncased-finetuned-sst-2-english")

classifier = load_pipeline()

st.title("💬 Sentiment Analyzer")
tab1, tab2 = st.tabs(["✍️ Single Text", "📁 Batch CSV"])

with tab1:
    text = st.text_area("Enter text to analyze", height=150,
                       placeholder="Type a review, tweet, comment...")
    if st.button("Analyze", type="primary") and text:
        result = classifier(text)[0]
        col1, col2 = st.columns(2)
        col1.metric("Sentiment", result['label'])
        col2.metric("Confidence", f"{result['score']:.1%}")
        
        color = "#22c55e" if result['label'] == "POSITIVE" else "#ef4444"
        st.progress(result['score'])

with tab2:
    csv_file = st.file_uploader("Upload CSV with a 'text' column", type=["csv"])
    if csv_file:
        df = pd.read_csv(csv_file)
        if "text" not in df.columns:
            st.error("CSV must have a 'text' column.")
        else:
            with st.spinner(f"Analyzing {len(df)} rows..."):
                results = classifier(df["text"].tolist(), truncation=True)
                df["label"]      = [r["label"] for r in results]
                df["confidence"] = [r["score"]  for r in results]
            
            # Summary charts
            col1, col2 = st.columns(2)
            with col1:
                fig = px.pie(df, names="label", title="Sentiment Split",
                            color_discrete_map={"POSITIVE":"#22c55e", "NEGATIVE":"#ef4444"})
                st.plotly_chart(fig, use_container_width=True)
            with col2:
                fig2 = px.histogram(df, x="confidence", color="label",
                                   title="Confidence Distribution")
                st.plotly_chart(fig2, use_container_width=True)
            
            st.dataframe(df)
            st.download_button("⬇️ Download", df.to_csv(index=False),
                              "sentiment_results.csv")

Tips, Gotchas & Best Practices

🔴 Common Mistakes to Avoid

⚠️
Loading models without caching. Always use @st.cache_resource. Without it, your 500MB model loads on every slider move.
⚠️
Putting st.set_page_config() after other st calls. It must be the first Streamlit call in the file, full stop.
⚠️
Blocking the main thread. Long inference (>2s) should use st.spinner() so users know work is happening. For very long jobs, look into st.status().

✅ Production-Ready Tips

💡
Use st.session_state to store prediction history across interactions — great for showing "last 5 predictions" or multi-step workflows.
💡
Use requirements.txt to pin your dependencies. Streamlit Cloud reads this automatically for deployment.
💡
Add a .streamlit/config.toml to customize theme colors, font, and whether the toolbar shows.
.streamlit/config.toml
[theme]
primaryColor     = "#38bdf8"
backgroundColor  = "#0b0f1a"
secondaryBackgroundColor = "#111827"
textColor        = "#e2e8f0"

[server]
maxUploadSize = 200   # MB — important for large images
