Module 01

Python Environment Setup

Your journey into Machine Learning begins with a stable foundation. We will configure Python 3.10+, manage dependencies, and fix common errors.

Preparation

Core Prerequisites

  • Python 3.10 or higher (3.11 recommended)
  • Administrator access to your machine
  • Stable internet connection
  • A terminal (CMD, PowerShell, or Bash)
Installation: OS

Windows Setup

Download the installer from python.org.
CRITICAL: Check the box "Add Python to PATH" during installation.

Installation: OS

macOS Setup

Use Homebrew for the cleanest management:

TERMINAL
brew install python@3.11
Verification

Testing The Install

Run these commands to ensure everything is working correctly:

TERMINAL
python --version
pip --version
Management

Virtual Environments

Why use venv? It prevents "Dependency Hell"—where different projects require conflicting versions of the same library.

[Image showing isolated python virtual environments vs a messy global environment]
Workflow

Create The Environment

Navigate to your project folder and run:

TERMINAL
python -m venv ml_internship
Activation

Activating on Windows

CMD / POWERSHELL
.\ml_internship\Scripts\activate

You should see (ml_internship) appear in your terminal prompt.

Activation

Activating on macOS/Linux

BASH / ZSH
source ml_internship/bin/activate
Package Management

The PIP Workflow

PIP installs libraries from the Python Package Index (PyPI).

TERMINAL
pip install numpy pandas
pip list # See what is installed
Best Practices

Requirements Files

Standardize your environment across different machines:

TERMINAL
pip freeze > requirements.txt
pip install -r requirements.txt
Troubleshooting

When Things Break

Environment errors are the #1 hurdle for beginners. Let's solve the common ones.

Error 01

"Python is not recognized"

Cause: Python was not added to your System PATH during installation.

Fix: Re-run the installer, choose "Modify," and ensure "Add to Environment Variables" is checked.

Error 02

"Permission Denied"

Fix for Windows: Run PowerShell as Administrator.
Fix for macOS/Linux: Use virtual environments (avoid sudo pip).
Error 03

"ModuleNotFoundError"

This happens if you install a library globally but try to run it inside a virtual environment (or vice versa).

Fix: Activate your venv FIRST, then run pip install.
IDE Config

Linking to IDEs

Visual Studio Code and PyCharm need to be told to use your ml_internship folder as the "Python Interpreter."

[Image showing VS Code python interpreter selection menu]
Maintenance

Leaving The Environment

When you are finished working, simply run:

deactivate

This returns you to your global system Python.

Verification

Final Checklist

  • Python 3.10+ Installed
  • Virtual Environment Created
  • pip install works without errors
  • Environment activated successfully
Pro Tip

Use .gitignore

NEVER upload your venv folder to GitHub. It is too large and specific to your machine.

Add venv/ or ml_internship/ to your .gitignore file.
