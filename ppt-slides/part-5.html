Module 05

Data & Learning Assets

Data is the fuel for Machine Learning. Today we learn how to source high-quality datasets and where to turn when you need to deepen your knowledge.

Overview

Finding the Fuel

A great ML engineer knows that data doesn't just "appear." You must know where to find public repositories and how to handle different file formats.

Sources

Kaggle: The ML Home

The world's largest data science community. Thousands of free datasets and competitions to test your skills.

Pro Tip: Look for "Silver" or "Gold" medal datasets; they usually have the best documentation.
Sources

Kaggle via Terminal

You can download datasets directly to your project using the command line:

BASH
pip install kaggle
kaggle datasets download -d username/dataset-name
Sources

Google Dataset Search

Think of it as "Google for Data." It aggregates datasets from across the web, including government portals and universities.

Sources

UCI ML Repository

The "Old Faithful" of the ML world. Home to classic datasets like Iris and Wine, perfect for testing your first algorithms.

Data Formats

The CSV Standard

Comma-Separated Values are the most common format. Pandas makes loading them a one-liner:

import pandas as pd
df = pd.read_csv('filename.csv')
Data Formats

JSON for Web Data

Common for APIs and nested data. Pandas can flatten these structures for you:

df = pd.read_json('data.json')
Data Formats

Excel & Python

Many business datasets live in .xlsx files. You can load specific sheets directly:

df = pd.read_excel('sales.xlsx', sheet_name='Q4')
